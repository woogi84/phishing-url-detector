# Phishing URL Detector - 설계 문서

## 1. 시스템 아키텍처

```
사용자 입력 (URL)
       ↓
   app.py (Streamlit UI)
       ↓
 detector/features.py     ← URL 파싱 및 특징 추출
       ↓
 detector/scorer.py       ← 가중치 기반 점수 계산
       ↓
  결과 출력 (점수 + 위험 요소 목록)
```

---

## 2. 탐지 특징 목록 (12가지)

| 특징 | 설명 | 가중치 |
|------|------|--------|
| url_length | URL 길이 75자 초과 | +10 |
| has_ip | IP 주소 직접 사용 | +25 |
| has_https | HTTPS 미사용 | +10 (없으면 -10) |
| dot_count | 점(.) 4개 이상 | +8 |
| hyphen_count | 하이픈(-) 2개 이상 | +8 |
| at_symbol | @ 기호 포함 | +20 |
| double_slash | 이중 슬래시(//) | +15 |
| subdomain_count | 서브도메인 2개 이상 | +10 |
| suspicious_keywords | 의심 키워드 포함 | +15 |
| is_shortened | URL 단축 서비스 사용 | +12 |
| special_char_count | 특수문자 3개 이상 | +8 |
| digit_in_domain | 도메인에 숫자 2개 이상 | +5 |

---

## 3. 위험도 기준

| 점수 | 등급 | 색상 |
|------|------|------|
| 70점 이상 | 위험 | 빨강 |
| 40~69점 | 주의 | 주황 |
| 39점 이하 | 안전 | 초록 |

---

## 4. 기술 스택 선정 이유

| 기술 | 선정 이유 |
|------|----------|
| Python | 문자열 처리 및 정규식 활용에 최적 |
| Streamlit | 별도 프론트엔드 없이 빠른 웹 UI 구성 가능 |
| tldextract | 도메인/서브도메인 파싱 정확도 높음 |

---

## 5. 프로토타입 v1 한계 및 개선 예정 사항

- WHOIS 기반 도메인 나이 조회 미구현 (v2 반영 예정)
- 실제 피싱 DB와 대조 기능 없음
- 단축 URL 원본 추적 미구현
