# phishing-url-detector 프로젝트 계획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Phishing URL Detector |
| 개발 도구 | Claude (Anthropic) – 바이브코딩 AI |
| 적용 프로세스 | 프로토타이핑 모델 |

---

## 2. 프로세스 모델 - 프로토타이핑 모델 
```
요구분석 → 프로토타입 개발 → 프로토타입 평가 → 개선 → 구현 → 인수/설치
```
### 정의
요구사항에 대한 피드백을 받기 위해 시스템을 실험적으로 만들어 사용자에게 보여주고 평가하게 하는 방법.

### 프로토타이핑 도구
-화면 생성기: 시스템의 작동을 시뮬레이션하여 사용자가 볼 수 있는 반응을 보여줌
-공동의 참조 모델: 사용자와 개발자의 의사소통을 도와주는 매개체

---

## 3. 본 프로젝트에 프로토타이핑을 적용한 이유
-피싱 URL 탐지의 정확한 판단 기준이 초기에 불투명하여 요구사항 확정의 어려움
-바이브 코딩을 활용한 빠른 프로토타입 생성이 가능하여 사이클 반복에 적합
-위험도 점수 기준, UI 구성 등을 사용자 평가를 통해 점진적으로 확정할 필요가 있음

---

## 4. 개발 단계별 계획


#### PHASE 1. 요구분석 
-탐지해야 할 URL 위협 유형 정의
-필요한 기능 목록 작성
-GitHub Issues 등록으로 요구사항 관리

요구사항 목록
-URL 길이, IP 직접 사용, HTTPS 여부 분석
-의심 키워드 탐지 (login, verify, account 등)
-위험도 점수 산출 (0~100)
-웹 UI에서 결과 시각화
-위험 요소별 상세 설명 제공


#### PHASE 2. 프로토타입 v1 개발 
-바이브코딩을 활용하여 핵심 기능 빠르게 구현
-결과물: 기본 URL 분석 + 점수 출력 화면

#### PHASE 3. 프로토타입 v1 평가 
-실제 피싱 URL / 정상 URL로 테스트
-평가 항목:
  - 위험도 점수가 직관적으로 이해되는가?
  - 탐지 누락이 있는가?
  - UI가 사용하기 편한가?
-평가 결과를 GitHub Issue로 등록 예정


#### PHASE 4. 프로토타입 v2 개선 
-v1 평가에서 도출된 문제점 반영
-AI와의 대화를 통해 수정 사항 구현


#### PHASE 5. 최종 구현 및 테스트 
-프로토타입 확정 후 코드 정리 및 안정화
-테스트 케이스 실행
-테스트 결과 문서화


#### PHASE 6. 인수 및 보고서
-SW 동작 캡처 (스크린샷)
-프로세스 적용 정리
-보고서 작성
